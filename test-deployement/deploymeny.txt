            il faudra l'ajouter (parfois possible via "Edit
            Pod" sans recréer le pod entièrement).----------
    Bien identifié. Effectivement, le port 8000 n'a pas été exposé à la création du pod — il faut corriger ça. Bonne nouvelle : sur RunPod, ce n'est généralement pas nécessaire de recréer le pod entièrement, on peut souvent éditer les ports exposés sur un pod existant.
    Solution 1 — Éditer le pod existant pour ajouter le port 8000 (à essayer en premier)
        1. Va sur l'interface RunPod, page "Pods"
2. Sur ton pod actif, cherche un bouton "Edit Pod" ou
   l'icône en forme de crayon ⚙️ / "..." à côté du pod
3. Cherche la section "Expose HTTP Ports" ou
   "TCP/HTTP Ports"
4. Ajoute 8000 à la liste des ports exposés
   (en plus du 8888 déjà présent pour Jupyter)
5. Sauvegarde / Applique les changements
(1. Clique dans le champ "Expose HTTP Ports"
2. Place ton curseur juste après "8888"
3. Tape : ,8000
   (virgule collée, puis 8000, sans espace)
4. Le champ doit afficher : 8888,8000
5. Sauvegarde / Applique les changements)

[ATTENTION] Sur certains types de pods (notamment Community
            Cloud), modifier les ports exposés nécessite un
            REDÉMARRAGE du pod. Si c'est le cas :
            - Le serveur Python qui tourne sera arrêté
            - Mais le modèle reste sur le disque (déjà
              transféré, donc pas à refaire)
            - Il faudra juste relancer "python server.py"
              après le redémarrage

Une fois le pod redémarré : relance le server
